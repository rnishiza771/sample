GitHub Pagesを使ったWebサイト公開手順

  1. GitHubアカウントを作成
    • GitHub公式サイトにアクセス
    • ユーザー名、メールアドレス、パスワードを入力し、アカウントを作成
    • メール認証を行い、アカウントの有効化
  2. リポジトリを作成
    • GitHubにログイン後、右上の「＋」→「New repository」を選択
    • Repository nameを入力（例：my-website）
    • 「Public」を選択し、「Add a README file」にチェック
    • 「Create repository」をクリック
  3. Webサイト用のファイルを追加
    • リポジトリ内で「Add file」→「Create new file」を選択
    • ファイル名にindex.htmlを指定
    • HTMLコードを入力し、「Commit new file」で保存
  4. GitHub Pagesの有効化
    • リポジトリの「Settings」タブをクリック
    • 左メニューから「Pages」を選択
    • 「Source」でmainブランチと/(root)を選び「Save」
    • 上部に表示されたURL（例：https://username.github.io/my-website）でアクセス可能
  5. 公開サイトを確認
    • ブラウザで表示されたURLにアクセスし、サイトが表示されることを確認

関連用語解説

GitHub
ソースコードを管理する場所（今はGitHub≒gitと認識してOK）。
リポジトリ（Repository）
GitHubの機能。ソースコードを管理する箱のこと。プロジェクトごとに分けて管理するイメージ。
コミット（Commit）
変更をリポジトリに反映すること。変更履歴の一区切りを作る作業。
コミット履歴（Commit History）
これまで行ったコミット（変更記録）の履歴。
ブランチ（Branch）
変更を分岐させるための仕組み。ブランチを切って異なる開発や修正を並行して行う。
github flow
シンプルなブランチ運用手法の一つ。メインブランチと機能ブランチを使って開発を進める。
マージ（Merge）
ブランチを統合すること。一方のブランチでの変更をメインや別のブランチに取り込む。
プルリクエスト（Pull Request）
ブランチの変更をメインブランチなどにマージするよう依頼する機能。レビューや承認のプロセスを踏む。
コンフリクト（競合）
マージ時に変更内容がぶつかること。自動でマージできない場合は手動で調整が必要。
markdown
HTMLより簡易的に書式が記述できる言語。
を書く必要なく、改行を入れるだけで改行など、簡易な記法が可能。
GitHub Pages
GitHubの便利機能。リポジトリで管理しているHTMLをそのままウェブサイトとして表示してくれる。
